## 📊 Features Overview

### Dashboard Features
- **4 KPI Cards:** Omzet, Order, Customer, Avg Order
- **8 Report Types:** Daily sales, top products, category distribution, sales performance, order funnel, low stock, pending orders, sales by person
- **AI Insights:** Generated by n8n + LLM
- **Anomaly Detection:** Automatic alerts with severity levels
- **Interactive Charts:** Line, Bar, Pie, Funnel (Recharts)
- **Real-time Filters:** Date range, sales filter

- **Product Management:** CRUD with stock indicators (Kritis/Rendah/Aman)
- **Customer Management:** CRUD with contact info
- **User Management:** CRUD with role assignment (Admin/Manager/Sales)
- **Telegram Settings:** Bot token, chat ID, schedule, threshold

### UX Features
- **Search:** Built into all data tables
- **Pagination:** 10 items per page
- **Loading States:** Spinner + text during async operations
- **Error Handling:** Clear error messages with retry options

- **Responsive Design:** Mobile, tablet, desktop optimized
- **Dark Theme:** Consistent dark UI throughout

## 📝 Notes for Backend Team

### Required Backend Endpoints
All endpoints listed in "API Endpoints Used" section must be active and return data in the expected format.

### Expected Response Format
```json
{
  "success": true,
  "message": "Success message",
  "data": { /* actual data */ }
}
```

### Error Response Format
```json
{
  "success": false,
  "message": "Error message in Bahasa Indonesia",
  "errors": { "field": "detail" }
}
```

### CORS Configuration
Backend must allow requests from `http://localhost:3000` (development) and production domain.

### JWT Token
- Token should be returned in `data.token` or `token` field
- Token should be valid for at least 24 hours
- Token should include user info (id, name, email, role)
